Type on the phone, watch the Kiro CLI echo every keystroke on your laptop. Same PTY, both screens. Run the IDE bridge in parallel and flip between tabs on the phone — one project, two surfaces.

How it works

Two small Node servers, one phone, zero cloud. Everything stays on your Wi-Fi.

🖥️

Laptop

Kiro IDE (Electron) or Kiro CLI (ConPTY) runs normally.

🔁

Bridge

CDP for the IDE (port 3000). PTY for the CLI (port 3001+). OTP-gated, HTTP + WebSocket.

📱

Phone

Any browser. Tabs for IDE and CLI. xterm.js renders the CLI, the bridge UI drives the IDE.

What's in the box

Single-use OTP

A 6-digit code prints to the terminal on every boot. One device per server lifecycle. Cookies are HttpOnly, session-scoped.

Scrollback replay

Late joiners see the current screen state, not a blank terminal. 256 KiB rolling buffer for the CLI bridge.

Multi-client

Open on laptop + phone at once. Effective PTY size is the minimum of all viewers, so nobody sees wrapping garbage.

Mobile-tuned UI

Sticky Esc/Tab/Ctrl-C/arrow row. visualViewport-aware re-fit so the keyboard doesn't scroll the terminal sideways.

Port-independent

IDE bridge on 3000, CLI bridge on 3001 by default. Change with --port or $PORT.

No cloud, no data leaves your LAN

Just Node + Express + ws + node-pty. Your Kiro session never touches a third-party service.

Quickstart

# Terminal 1 — IDE bridge
kiro --remote-debugging-port=9000
npx kiro-mobile-bridge

# Terminal 2 — CLI bridge
cd kiro-cli-mobile-bridge
npm install
npm start

# Phone
# open http://<laptop-ip>:3000 and http://<laptop-ip>:3001
# enter the 6-digit OTP printed in each terminal
